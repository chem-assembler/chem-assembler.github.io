# 反応機構アニメーション 設計書（P5-5）

開発コンセプト（DEVELOPMENT.md 1.1章-2）「巻矢印による電子の動き・結合の組み換えをアニメーションで再現し、反応機構の学習をサポートする」を実現するための基盤設計。実装前の合意用ドキュメントであり、実装の進行に合わせて更新する。

---

## 1. 目的と体験イメージ

- 紙の上に書く反応機構（構造式＋巻矢印）と同じ見た目を、そのまま動かして見せる。立体的な軌道表現はしない（コンセプト1.1-1の平面作図主義を維持）。
- 学習者の体験は2段階：
  1. **観察モード（本設計のスコープ）**: 反応をステップ再生で眺める。巻矢印が描かれ、結合が組み変わり、分子が変形する。
  2. **予測モード（将来・M4）**: 「次に何ができる？」をユーザーが既存パズルUIで組み立て、`verifyMolecule` で判定する。既存パズルとの自然な統合点。

## 2. スコープ（対象反応）

高校〜大学初級の代表反応から開始する：

| 分類 | 例 | 矢印の種類 |
|---|---|---|
| 付加反応 | エテン + Br₂ | 巻矢印（電子対・両羽） |
| 置換反応 | メタン + Cl₂（ラジカル連鎖） | 片羽矢印（単電子） |
| エステル化 | 酢酸 + エタノール | 巻矢印・多段 |
| 芳香族置換 | ベンゼンのニトロ化 | 巻矢印・中間体あり |

芳香環はケクレ構造固定で描く（検証側のケクレ吸収は verifyMolecule のみの責務。描画には影響しない）。

## 3. データモデル

### 3.1 方針

- **反応 = 分子状態（スナップショット）の列 + 状態間を繋ぐステップ（巻矢印の集合）**として表現する。
- 分子状態は stages.json の `target` と同じ `{atoms, bonds}` 形式を再利用し、`createTargetFromData` 相当のローダーを流用する。
- 原子の同一性は **states 間で atoms 配列の index を共通IDとして扱う**。原子は途中で消さない（脱離した原子・分子も座標移動で画面内に描き続ける）。新たに登場する分子（求電子剤など）は state 0 から画面端に置いておく。

### 3.2 reactions.json スキーマ（新設ファイル）

```json
[
  {
    "name": "エテンへの臭素の付加",
    "series": "付加反応",
    "desc": "π結合の電子対が求電子剤Br₂を攻撃し…",
    "states": [
      { "atoms": [{"element":"C","x":360,"y":300}, ...], "bonds": [{"atom1Index":0,"atom2Index":1,"type":2}, ...] },
      { "atoms": [...], "bonds": [...] }
    ],
    "steps": [
      {
        "from": 0,
        "to": 1,
        "caption": "C=Cのπ電子対がBrを攻撃し、Br-Br結合の電子対はBr⁻へ移る",
        "arrows": [
          { "source": {"type":"bond", "atoms":[0,1]}, "target": {"type":"atom", "index":2}, "style":"pair" },
          { "source": {"type":"bond", "atoms":[2,3]}, "target": {"type":"atom", "index":3}, "style":"pair" }
        ]
      }
    ]
  }
]
```

- `arrows[].source` / `target`: `{"type":"bond","atoms":[i,j]}`（結合の中点）または `{"type":"atom","index":i}`（原子＝非共有電子対や求核中心）。
- `arrows[].style`: `"pair"`（両羽の巻矢印＝電子対）| `"single"`（片羽＝単電子、ラジカル用）。
- `steps[].from / to`: states のインデックス。1ステップ = 1遷移。中間体は states を増やして表現する。

### 3.3 既存モデルへの拡張

- `Atom` に `charge`（形式電荷: +1/-1/0）と `partialCharge`（"δ+"/"δ-"/null、表示専用）を追加。**検証ロジック（verifyMolecule）には当面使わない**（パズル側の非互換を作らないため）。
- **反応モードでは自動水素補完（calculateHydrogens）を無効化**し、動く水素（プロトン移動など）は states に明示原子として記述する。パズルモードの挙動は変えない。

## 4. 描画・アニメーション

### 4.1 巻矢印（静的描画）

- SVG `<path>` の3次ベジェ曲線。始点・終点は source/target の座標（結合中点 or 原子円の縁）。制御点は始点終点の中点から法線方向に `曲率オフセット`（基本 30px、矢印同士の重なり回避で符号反転可）。
- 矢尻は `<marker>` で定義（`pair`=通常の三角矢尻、`single`=片側だけの羽）。色はネオンピンク系（既存UIの `--neon-pink`）で構造式と区別する。
- 専用グループ `<g id="arrows-group">` を bonds/atoms の上、ui-group の下に追加。

### 4.2 ステップ再生アニメーション

1フェーズ2段階、`requestAnimationFrame` のみで実装（外部ライブラリ禁止: 開発方針2章）：

1. **矢印フェーズ**（~0.8s）: `stroke-dasharray`/`dashoffset` で巻矢印が「描かれていく」。
2. **遷移フェーズ**（~1.0s）: from状態→to状態へ、
   - 原子座標は線形補間（index対応で同一原子を移動）
   - 消える結合はフェードアウト、生じる結合はフェードイン（透明度補間）
   - 結合次数の変化（2→1など）は新旧の線をクロスフェード
3. フェーズ完了後、キャプションを更新して次ステップ待ち。

### 4.3 実装配置

- `reaction.js`（新設）: `ReactionPlayer` クラス。states のロード、補間、矢印描画、再生制御。DOMに依存する描画はここに集約。
- `chemistry.js`: `Atom.charge` 追加のみ（純粋ロジック維持: 開発方針2章）。
- `game.js`: モード切替（パズル ⇄ 反応機構）のUI結線のみ追加。`renderAtom`/`renderBond` は座標引数で再利用できる形なので ReactionPlayer から流用する。

## 5. UI

- 右パネルに「モード」切替（構造パズル / 反応機構）。反応機構モードでは：
  - シリーズ・反応選択ドロップダウン（stages と同じUIパターン）
  - コントロールバー: ⏮ 前ステップ / ▶ 再生 / ⏭ 次ステップ / 🔄 最初から
  - キャプション表示（steps[].caption）
- 反応機構モード中はパズル系ツール（配置・結合・消しゴム）を無効化。キャンバスのパン・ズームは共用。

## 6. 実装マイルストーン（ロードマップに反映）

- **M1**: `reactions.json` スキーマ確定＋ローダー＋state 0 の静的表示＋巻矢印の静的描画（エテン+Br₂ 1本）
- **M2**: ステップ再生（矢印描画アニメ＋状態補間アニメ＋コントロールUI）
- **M3**: 代表反応データ 3本追加（ラジカル置換・エステル化・ニトロ化）と片羽矢印対応
- **M4**: 予測モード（次状態をパズルUIで組み立て → `verifyMolecule` 判定）

## 7. リスク・未決事項

- **プロトン移動の見せ方**: 自動水素をオフにする境界（モード切替時）の描画整合に注意。
- **矢印の重なり**: 多段反応で矢印が交差する場合の曲率自動調整は M2 では手動指定（`curvature` オプション追加）で逃げる。
- **形式電荷の描画位置**: 原子ラベル右上に小さく表示（不斉マーク * と同じ手法）。不斉マークと重なる場合は左上へ退避。
- **立体模型対照（P5-6）との関係**: 本設計は平面表現に徹する。P5-6 は別ビューとして検討し、本基盤には持ち込まない。
