# B群: 酸化還元モードの設計（2026-07-23）

ユーザー方針: **半反応式だけをアニメーション化し、それらを任意の整数倍して足し合わせる**。
半反応式が「部品」、倍率合わせが「組み立て」。e⁻ の数合わせが本モードの核心。

## 学習ループ（イオン反応モードと同型）

1. **半反応式が部品として提示される** — [酸化] Zn → Zn²⁺＋2e⁻ ／ [還元] Ag⁺＋e⁻ → Ag
2. **倍率 ×a・×b を操作** — ビーカー内の配置（板の上の原子 a 個・溶液中のイオン b 単位）が即座に連動
3. **「▶ 反応を見る」** — 酸化: 板の原子が e⁻ を出してイオンになり溶け出す →
   還元: 溶液のイオンが板へ泳ぎ、板の e⁻ を受け取って原子になり析出（金属樹）／H₂ の泡になる
4. **失敗の表現**（発見学習）
   - e⁻ が余る → 板の上で赤く点滅「e⁻ が n 個余った。受け取る相手が足りない」
   - e⁻ が不足 → イオンが板の前で待ちぼうけ「e⁻ が足りない」
5. **ぴったり** → 足し合わせ表示: `Zn ＋ 2Ag⁺ ＋ ~~2e⁻~~ → Zn²⁺ ＋ 2Ag ＋ ~~2e⁻~~`
   と e⁻ を打ち消し線で消し、まとまったイオン反応式を提示。倍率＝係数の実感
6. クリア条件 = アニメがぴったり完了 ＋ 倍率が最簡整数比

## モデル（model.js に追加・純ロジック）

- 化学種: Zn / Zn²⁺ / Cu / Cu²⁺ / Ag / H₂ / **e⁻**（atoms 空・charge −1 の特別種）
- `HALF_REACTIONS`: { disp, left: [{sp,n}], right: [{sp,n}], kind: oxidation|reduction }
  - 半反応式自体が原子・電荷保存を満たすことをテストで機械検証
- `REDOX_STAGES`: { ox, red, answer: [a,b] }
- `checkRedoxMultipliers(stage, a, b)`: e⁻ 授受が等しい ＆ gcd(a,b)=1
- `combineHalves(stage, a, b)`: 両辺を倍率つきで合算し、両辺に現れる種（e⁻）を打ち消して
  イオン反応式を返す。結果が compareSides で釣り合うことをテストで検証

## 画面（redox.html + redox.js。model.js と style.css を共用）

- index.html とヘッダーで相互リンク（モード切り替え）
- ステージ: r1 Zn×Cu²⁺（1:1）／ r2 Cu×Ag⁺（1:2 倍率合わせの本番）／ r3 Zn×2H⁺（H₂ 泡）
- ビーカー左に金属板。e⁻ は黄色の小粒として板の上に「たまる」→ 受け手へ飛ぶ
- 検証はイオン反応モードと同じ思想: 個数（e⁻ プール・待ちイオン・析出数）のみで判定、
  座標は見た目専用。`window.RedoxEq.advance(ms)` フックで UI テスト駆動

## 将来（このエンジンの延長）

- イオン化傾向シリーズ（どちらが溶けてどちらが析出するかの予想モード）
- B3 電池: 2枚の板と導線で e⁻ を回す（半反応式部品はそのまま流用）
- 半反応式そのものを組み立てる練習（e⁻ の数を自分で決める）

## 溶液中の酸化還元（redox-solution・2026-07-24 追加設計）

金属樹（`redox-metal`）は「板＋溶液イオン」前提。KMnO₄・K₂Cr₂O₇ などは**酸化剤も還元剤も溶液中**で、
H⁺ が反応に関与し、**色変化**が最大の見どころ（酸化還元滴定の終点＝色が変わる/残る）。
[DESIGN_reaction_library.md](DESIGN_reaction_library.md) の animationType レジストリで `redox-solution` として扱う。
**モデルは既存の枠（半反応式×倍率→e⁻ 打ち消し→イオン反応式）をそのまま流用**し、拡張は (1) 半反応式データ
(2) 酸化数表 (3) 色データ、と**アニメが新規**の3点に閉じる。

### モデル拡張（model.js・純ロジック）

**HALF_REACTIONS 追加**（酸化剤側に H⁺ と H₂O が入る。原子・電荷保存＋e⁻ を含むことを既存テストで検証）:

| id | 半反応式 | kind | e⁻ |
|---|---|---|---|
| `MnO4_red` | MnO₄⁻ ＋ 8H⁺ ＋ 5e⁻ → Mn²⁺ ＋ 4H₂O | reduction | 5 |
| `Cr2O7_red` | Cr₂O₇²⁻ ＋ 14H⁺ ＋ 6e⁻ → 2Cr³⁺ ＋ 7H₂O | reduction | 6 |
| `Fe2_ox` | Fe²⁺ → Fe³⁺ ＋ e⁻ | oxidation | 1 |
| `H2O2_red` | H₂O₂ ＋ 2H⁺ ＋ 2e⁻ → 2H₂O | reduction | 2 |
| `H2O2_ox` | H₂O₂ → O₂ ＋ 2H⁺ ＋ 2e⁻ | oxidation | 2 |
| `oxalate_ox` | (COO)₂²⁻ → 2CO₂ ＋ 2e⁻ | oxidation | 2 |
| `I_ox` | 2I⁻ → I₂ ＋ 2e⁻ | oxidation | 2 |

代表 REDOX_STAGES（溶液中）: MnO₄⁻×Fe²⁺（1:5）、Cr₂O₇²⁻×Fe²⁺（1:6）、MnO₄⁻×(COO)₂²⁻（2:5）。
`checkRedoxMultipliers`/`combineHalves` は無改修で流用（e⁻ 数を合わせ最簡比、両辺共通種を打ち消し）。
※ combineHalves は現状 e⁻ しか打ち消さないが、溶液中では H⁺・H₂O は打ち消し対象にならない（片辺のみ）ため問題なし。
両辺に同種が残るケース（例外的に H₂O が両辺）が出たら打ち消し一般化を検討。

**OXIDATION 拡張**（「原子の酸化数合計＝種の電荷」を既存テストで機械検証）:
`MnO4-`={Mn:+7, O:−2}、`Mn^2+`={Mn:+2}、`Cr2O7^2-`={Cr:+6, O:−2}、`Cr^3+`={Cr:+3}、
`Fe^3+`={Fe:+3}、`H2O`={H:+1, O:−2}、`O2`={O:0}、`H2O2`={H:+1, O:−1}（**過酸化物の O=−1 は例外**）、
`CO2`={C:+4, O:−2}、`(COO)2^2-`={C:+3, O:−2}、`I-`={I:−1}、`I2`={I:0}。
- 多原子イオンの酸化数は O=−2・H=+1 を基準に逆算するのが通例だが、本アプリは**値を直接 OXIDATION に持つ**
  （逆算ロジックは持たない）。過酸化物のような例外もデータで表現でき、検証もデータ駆動で回る。
- `oxChangeOfHalf` は「変化する元素がちょうど1つ」を仮定。MnO₄⁻→Mn²⁺（Mn のみ変化）等は満たす。
  H₂O₂ の不均化（O が +?）など1元素の増減が非自明な反応は当面ステージにしない（要注意メモ）。

### 色データ（新規・見た目専用だが機械検証する）

種→色の表 `SPECIES_COLOR`（有色種のみ。無色は既定の淡色）:
`MnO4-`=赤紫、`Mn^2+`=ほぼ無色、`Cr2O7^2-`=橙、`Cr^3+`=緑、`Fe^2+`=淡緑、`Fe^3+`=黄褐、`I2`=褐。
- **粒の色**＝その種の色（房/円の塗り）。
- **溶液全体の色**＝現存する有色種の量で決まる混合色。滴定では「酸化剤（紫）が残るか」で終点を判定・演出。
- 検証: 半反応式・反応に登場する有色種すべてに色定義があること（データ駆動テスト）。

### アニメーション（redox.js を拡張 or 新 renderer）

`redox-metal`（板あり）と分岐。`redox-solution` は**板なし・両者とも溶液中の浮遊粒**:

1. 倍率 ×a・×b に応じて酸化剤 a 単位・還元剤 b 単位＋H⁺ を配置
2. 「▶ 反応を見る」で、還元剤が e⁻ を放ち（黄粒）、酸化剤が受け取る。酸化剤は H⁺ を取り込んで
   Mn²⁺＋H₂O 等に変化（房の作り替え）。**変化の瞬間に粒の色が遷移**（紫→無色 など）
3. **溶液全体の背景色**も反応進行で遷移。ぴったりで無色化（=終点）
4. 失敗表現は既存流用: e⁻ 余り＝赤点滅、不足＝待ちぼうけ。過剰の酸化剤＝**紫が残る**（滴定の実感）
5. ぴったり＋最簡比でクリア。足し合わせ→イオン反応式（H⁺・H₂O 込み）を表示

判定は既存思想どおり**個数のみ**（e⁻ プール・変化した粒数・残留有色種数）。色・座標は見た目専用。
`window.RedoxEq` フックに溶液色/残留有色種を足して UI テストで駆動する。

### reactions.json との接続

- 溶液中反応は `animationType:"redox-solution"`、`classes.redox:"溶液中の酸化剤還元剤"`。
- 現在は `playable:false`（参照エントリ。式・イオン反応式・解説・色の記述のみ）。
- 本エンジン実装後、対応ステージを `playable:true` にし、index の「▶遊ぶ」を redox.html?rxn=<id> に接続
  （redox.js にも ?rxn ディープリンク受けを追加。app.js と同型）。

### 段階

1. モデル拡張（HALF_REACTIONS＋OXIDATION＋SPECIES_COLOR）＋検証テスト（アニメなしでも回る）
2. `redox-solution` アニメ（まず MnO₄⁻×Fe²⁺ 1本）。色遷移・e⁻ 溶液内授受・H⁺ 取り込み
3. 該当 reactions.json を playable 化＋ redox.html ディープリンク
4. Cr₂O₇²⁻・(COO)₂²⁻・H₂O₂ を順次。滴定終点モード（過剰で紫残留）は 2 の後に磨く
