# 異性体の書き出し練習モード 設計書（P12-1）

分子式を提示し、ユーザーが構造異性体を1つずつ描いて登録していき、
「全部そろったか・重複していないか」をアプリが判定する練習機能。
2026-07-24 ユーザー要望（P12-1）に基づく。実装前の合意用ドキュメント。

---

## 1. 目的と学習価値

- 「C₄H₁₀O の異性体をすべて書け」は定期試験・入試の定番だが、紙では**自分の答えが
  重複していないか・網羅できたか**を自力で確かめられない。ここをアプリが即時判定する。
- 重複判定を通じて「**描き方が違っても同じ化合物**」（回転・裏返し・屈曲・ケクレ位相）を
  体感させる。フェーズ8「表記の同値性を学ぶ」の実践編にあたる。
- 既存の学習ビュー（📚異性体ギャラリー・書き出し方の4ステップ解説）を「答えを見る」側、
  本機能を「自分で書き出す」側として対にする。

## 2. 既存資産（すべて実装済み・流用）

| 資産 | 用途 |
|---|---|
| `enumerateConstitutionalIsomers(elements, hCount)`（chemistry.js） | 正解集合の生成（重原子6個まで） |
| `canonicalCode(mol)`（chemistry.js） | 登録済みとの重複判定・正解集合との照合 |
| `layoutMolecule` ＋ ギャラリー描画（learn.js） | 登録済み異性体のサムネイル表示 |
| `categorizeMolecule`（learn.js） | 官能基分類ヒント（アルコール◯種・エーテル◯種） |
| `computeMolecularFormula`・`splitMolecules`（game.js） | 分子式チェック・単一分子チェック |
| 学習モードの右パネル（P10） | 練習UIの置き場所 |

新規のアルゴリズムは**不要**。本機能はUIとゲームフローの追加のみ。

## 3. モード内の位置づけ

- **📚学習モードの一機能**とする（新トップレベルモードは作らない）。
  学習パネルに「✏️ 異性体の書き出し練習」カードを新設し、問題を選ぶと練習セッションが始まる。
- 反応機構ビューアと同様に「学習モードを離れたら練習セッションを終了」する。
- 作図は共通キャンバスをそのまま使う（描く道具・操作は通常と完全に同一。
  練習のために作図仕様を変えない）。

## 4. 出題

### 4.1 問題リスト（M1は固定リスト）

列挙エンジンの対応範囲（重原子≤6・列挙打ち切りなし）で、高校範囲の定番から段階的に:

| # | 分子式 | 異性体数 | ねらい |
|---|---|---|---|
| 1 | C₄H₁₀ | 2 | 入門（直鎖と枝分かれ） |
| 2 | C₅H₁₂ | 3 | 枝分かれの数え上げ |
| 3 | C₃H₈O | 3 | アルコールとエーテル |
| 4 | C₆H₁₄ | 5 | 骨格異性の網羅（主鎖の選び方） |
| 5 | C₄H₈ | 5 | 不飽和度1（アルケンの位置＋環） |
| 6 | C₄H₁₀O | 7 | 総合（アルコール4＋エーテル3） |

- 異性体数はデータに持たず**起動時に列挙エンジンから計算**する（数値の二重管理をしない。
  テストM1と同じ既知値で自己検証される）。
- 問題リストは learn.js 内の定数とする（数件なので新規JSONは作らない。増えたら再検討）。

### 4.2 幾何異性・立体異性の扱い

- 列挙・判定とも**構造異性（トポロジー）のみ**。2-ブテンのシス/トランスは1種と数える。
  問題カードに「シス・トランスや鏡像の区別は数えません」と明記する（現行の判定原則と一致）。

## 5. 練習のフロー

```
問題を選ぶ → 分子式と目標数を提示（例: C₄H₁₀O の異性体 全7種）
  ↓ キャンバスに1つ描く
「＋この分子を登録」ボタン
  ├─ 分子式が違う → 拒否トースト（いまの式を表示: C₄H₈O など）
  ├─ 分子が2個以上 → 拒否トースト（1分子ずつ登録）
  ├─ 登録済みと同型 → ★重複通知: 一致した登録サムネイルを点滅ハイライトし
  │   「登録済みの◯番『2-メチルプロパン』と同じ化合物です
  │   （描き方が違っても、つながり方が同じなら同一）」← 名称で同一性を示す
  └─ 新規 → 名称を判定してサムネイル＋名称ラベルをトレイに追加（n/全m 更新）、
      キャンバスを自動クリア
  ↓ 繰り返し
全種登録 → 完了演出（🎉）＋★答え合わせ一覧: 全異性体を系統順（7章の系列順）に
           名称付きで並べる ＋ 分類まとめ（アルコール4種・エーテル3種）＋ クリア記録
```

### 5.1 命名の役割（ユーザー要望 2026-07-24）

- **登録した構造には必ず名称を付けて見せる**。重複が起きたときに
  「別の絵だと思っていた2つが同じ名前」という形で同一性が腑に落ちる。
- 名称は既存の `lookupCompoundName`（compounds.json 照合）を使う。
  **前提整備**: 問題6問の全異性体（延べ25種）に名称が登録されていることを実装時に
  機械検証し、欠けは compounds.json に追補する（P12-3 と連動。テストにも組み込む）。

### 5.2 主鎖の図示 — 「適切な書き方」の教示（ユーザー要望 2026-07-24）

- 登録・重複通知の際、**ユーザーの作図が正準的な書き方でない場合**は、命名とあわせて
  **主鎖（最長の炭素鎖）を作図上にハイライト**して見せ、標準の書き方を理解させる。
- 「正準的でない」の検出: 作図上で**まっすぐ（ほぼ一直線・L字含む直交の連続）に見えている
  炭素列**と、実際の最長炭素鎖が**一致しない**とき（例: 4個を横一列に描いて枝に2個
  ぶら下げたが、実際の最長鎖は枝をまたぐ5個 — 命名の定番の落とし穴）。
- 表示: キャンバス上の該当原子・結合をオレンジでハイライト（不斉ハイライト P7-4 の流用）＋
  「この分子の主鎖はこの5個です（名称: 2-メチルブタン）。主鎖をまっすぐ描くと ↓」として
  標準レイアウトのサムネイル（layoutMolecule）を並置する。
- 実装: chemistry.js に `findLongestCarbonChain(mol)`（純粋関数・原子ID列を返す）を追加。
  環を含む分子は最長鎖の概念が別（環が主構造）なので、環検出時は環をハイライトして
  「環を基本骨格として描きます」と案内するにとどめる。

- **登録時のキャンバス自動クリアは saveState を積んでから**行う（↩で直前の作図に戻れる。
  登録トレイ自体は Undo の対象外＝練習の進行は巻き戻さない）。
- 途中でやめる: 「練習をやめる」ボタン＋学習モード離脱で終了。進行中セッションは破棄
  （クリア記録だけ localStorage に残す。ステージのクリア記録と同じ方式）。
- 正解集合との照合について: 分子式・価標・単一分子の検査を通った構造は原理的に必ず
  列挙集合に含まれるが、**防御的に照合し、万一不一致なら「判定できない構造」として
  console に記録**する（列挙エンジンの欠落検出を兼ねる。監査と同じ思想）。

## 6. UI（学習パネル内の練習カード）

- **問題選択**: カードに問題ボタンを列挙（`C₄H₁₀（2種）` … クリア済みには✓）。
- **練習中**: カードの表示を切り替え:
  - ヘッダ: `✏️ C₄H₁₀O の異性体 3/7`
  - 登録トレイ: サムネイル格子（ギャラリーと同じ描画。番号ラベル付き）
  - ボタン: `＋この分子を登録`（主ボタン）・`💡ヒント`・`練習をやめる`
- **ヒント（段階式・押すたびに1段階）**: ★未発見の構造から系統的に生成する
  （ユーザー要望 2026-07-24。「システマチックに全部書き出せる手順」を教えるのが目的）。
  1. **系列の内訳**: 未発見分を骨格系列ごとに数えて示す
     （例: 「主鎖5＋メチル基1つ の系列があと1つ / 主鎖4＋炭素2個分の側鎖 の系列があと2つ」）
  2. **系列内の書き出し手順**: 未発見構造の系列に対応する定型手順文を示す
     - 置換基・官能基の位置ずらし系: 「◯◯の付く位置を端から順に一通りずらして
       試しましたか（対称な位置は同じものになります）」
     - 炭素2個分の側鎖系: 「側鎖に炭素を2個使う置き方は3通り —
       **エチル基1つ / メチル基2つを同じ炭素に / メチル基2つを別の炭素に**」
     - 不飽和・環系: 「二重結合の位置ずらしと、環にする案の両方を数えましたか」
  3. **答えを見る**: 全異性体を系統順に並べた答え合わせ一覧（下記）を開く
- **答え合わせ一覧（完了時・ヒント3段階目で共通）**: 全異性体を**系統順に**並べて表示する。
  順序 = 主鎖の長い順 → 側鎖の付け方（エチル → メチル×2同一C → メチル×2別C の例示順）→
  官能基・多重結合の位置順。各項目は サムネイル＋名称＋発見済み✓/未発見（オレンジ枠）。
  「系列がすべて並んでいる」こと自体が書き出し方の教材になる。
  - 実装: 各異性体から系統キー `(最長炭素鎖長, 側鎖サイズ構成と同一炭素かの区別,
    官能基位置の正準番号)` を計算してソートする純粋関数を chemistry.js に追加
    （`isomerSeriesKey(mol)`）。ヒント1・2の系列分類もこのキーを共用する。
- **モバイル**: カードは右パネル（縦=下シート/横=ドロワー）内にそのまま載る。
  進行表示 `3/7` は化合物名チップ（P11-M3c）の隣に出すことも検討（M2）。

## 7. 実装方針

- **learn.js に IsomerPractice クラスを追加**（学習ビューの責務の範囲。
  ゲームフローの状態は practice インスタンスに閉じる。chemistry.js には手を入れない）。
- 状態: `{ problem, targetCodes: Map<code, iso>, found: Map<code, {mol, order}> }`。
  判定は `canonicalCode(userMolecule)` を targetCodes / found と突き合わせるだけ。
- index.html: 学習ブロックに練習カードの器（`data-modes="learn"`）とボタン類を追加。
- game.js への変更は最小限: 登録ボタンから `game.userMolecule` を読む・クリアする導線のみ
  （既存の verify/summon と同じ呼び方）。
- クリア記録: `localStorage: chemIsomerPractice.<分子式>` に完了フラグ。

## 8. マイルストーン

- **M1: コアループ** — 問題リスト・登録/重複/分子式チェック・**名称付き**トレイ表示・

- **M2: 学習性の強化** — 系統的な段階ヒント（系列内訳→書き出し手順→答え。
  `isomerSeriesKey` による系列分類）、主鎖の図示（`findLongestCarbonChain`・
  非正準な書き方の検出とハイライト＋標準レイアウト並置）、完了演出と分類まとめ。
- **M3: 出題の拡張** — 任意の分子式入力での練習（列挙可能な範囲かを検査して受け付ける）、
  P12-2（列挙エンジンの範囲拡張）と連動して問題を追加。

## 9. リスク・未決事項

- **列挙の負荷**: 問題リストの6問はいずれも既知の高速ケース（最大 C₄H₁₀O=13ms）。
  M3の自由入力では overflow（打ち切り）検出時に「この分子式は練習に対応していません」と断る。
- **「登録＝クリア」の誤操作**: 登録でキャンバスが消えることに驚かないよう、
  初回登録時に一度だけトースト（↩で戻せる旨）を出す。
- **エノール等の不安定構造**: 列挙エンジンは価標上有効な構造をすべて数える。C₄H₈ 等の
  問題では発生しないが、M3 の自由入力で「教科書では数えない構造」（エノール等）が
  正解集合に入る分子式があり得る。M3 で分類フィルタ（enol 除外オプション等）を検討。
- **回帰テストのモード前提**: 練習はモード状態を持つため、テストは setMode('learn') →
  練習開始 → 終了までを1テスト内で完結させ、他テストへ状態を漏らさない（reset で練習を
  強制終了する処理を追加）。
- **命名の網羅**: 6問の全異性体に compounds.json の名称が必要（M1で機械検証）。
  IUPAC命名アルゴリズムは未実装のため、当面はライブラリ照合方式。M3 の任意分子式では
  名称のない異性体が出るので「（名称未登録）」表示を許容し、P12-3（系統名）で解消する。
- **主鎖検出と作図の照合**: 「まっすぐ描かれた炭素列」の判定は直交作図（水平・垂直の連続）
  を前提にした幾何判定になる。座標は本来トポロジー判定に使わない原則（開発方針4章）だが、
  これは正誤判定ではなく**表示上の教育支援**なので、シス/トランス命名（4章-4）と同じ
  例外扱いとする。
